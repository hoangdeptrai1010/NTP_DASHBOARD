## 2. Database — Oracle

### 2.1 Sơ đồ bảng hiện tại (đã có trong SQL)

**Nhóm nghiệp vụ viện phí:**
```
DMNHOMVIENPHI → DMLOAIVIENPHI → DMGIAVIENPHI
DMDOITUONG
DMKHOAPHONGBV
BENHNHAN
VIENPHI (FK: BENHNHAN, DMKHOAPHONGBV)
VIENPHICT (FK: VIENPHI, DMDOITUONG, DMGIAVIENPHI, DMKHO)
VIENPHICTTHUOC (FK: VIENPHI, M_DMBD, DMKHO)
```

**Nhóm kho thuốc/vật tư:**
```
DMNHOMKHO → DMKHO
M_DMBD (FK: DMNHOMKHO)
```

**Nhóm auth & hệ thống (thêm mới so với ERD):**
```
APP_ROLES → APP_USERS (FK: APP_ROLES, DMKHOAPHONGBV)
APP_USERS → BACSI
APP_USERS → TRUONGKHOA
APP_USERS → GIAMDOC
APP_ROLES → QUYEN
APP_USERS → DASHBOARD_LAYOUT_PREFS
```

### 2.2 Gap cần vá (phát hiện khi so ERD vs SQL)

| Vấn đề | Bảng bị ảnh hưởng | Cách xử lý |
|--------|-------------------|------------|
| Thiếu `DMTHUOC` (danh mục thuốc gốc) | `M_DMBD` cần FK vào đây | Thêm bảng `DMTHUOC(IDTHUOC, TENTHUOC, HOATCHAT, DONVITI)` |
| `VIENPHICT` thiếu `IDKHOAPHONG` | Không filter được CT theo khoa | `ALTER TABLE VIENPHICT ADD IDKHOAPHONG NUMBER` |
| `VIENPHI` thiếu nguồn thanh toán | Không vẽ được biểu đồ BHYT vs tự trả | `ALTER TABLE VIENPHI ADD NGUON_THANHTOAN VARCHAR2(30)` |
| `BENHNHAN` thiếu thông tin BHYT | Không phân tích BHYT | Thêm `SO_BHYT`, `LOAI_BHYT`, `CMND` |
| Nhiều bảng thiếu `CREATED_AT` | Không audit được lịch sử | Thêm `CREATED_AT TIMESTAMP DEFAULT CURRENT_TIMESTAMP` |

### 2.3 SQL bổ sung cần chạy

```sql
-- 1. Bảng danh mục thuốc gốc
CREATE TABLE DMTHUOC (
    IDTHUOC     NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    TENTHUOC    VARCHAR2(255) NOT NULL,
    HOATCHAT    VARCHAR2(255),
    DONVITI     VARCHAR2(50),
    CREATED_AT  TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

ALTER TABLE M_DMBD ADD IDTHUOC NUMBER;
ALTER TABLE M_DMBD ADD CONSTRAINT FK_BD_THUOC
    FOREIGN KEY (IDTHUOC) REFERENCES DMTHUOC(IDTHUOC);

-- 2. VIENPHICT thêm khoa
ALTER TABLE VIENPHICT ADD IDKHOAPHONG NUMBER;
ALTER TABLE VIENPHICT ADD CONSTRAINT FK_CT_KHOA
    FOREIGN KEY (IDKHOAPHONG) REFERENCES DMKHOAPHONGBV(IDKHOAPHONG);

-- 3. VIENPHI thêm nguồn thanh toán
ALTER TABLE VIENPHI ADD NGUON_THANHTOAN VARCHAR2(30);
-- Giá trị: 'BHYT', 'TU_TRA', 'MIEN_GIAM', 'HO_TRO'

-- 4. BENHNHAN thêm thông tin BHYT
ALTER TABLE BENHNHAN ADD SO_BHYT    VARCHAR2(20);
ALTER TABLE BENHNHAN ADD LOAI_BHYT  VARCHAR2(30);
ALTER TABLE BENHNHAN ADD CMND       VARCHAR2(20);
-- LOAI_BHYT: 'TOAN_PHAN', 'MOT_PHAN', 'KHONG'

-- 5. Audit timestamps
ALTER TABLE BENHNHAN    ADD CREATED_AT TIMESTAMP DEFAULT CURRENT_TIMESTAMP;
ALTER TABLE VIENPHI     ADD CREATED_AT TIMESTAMP DEFAULT CURRENT_TIMESTAMP;
ALTER TABLE VIENPHICT   ADD CREATED_AT TIMESTAMP DEFAULT CURRENT_TIMESTAMP;

-- 6. Index bổ sung cho dashboard query
CREATE INDEX IDX_CT_KHOA       ON VIENPHICT(IDKHOAPHONG);
CREATE INDEX IDX_CT_NGAY       ON VIENPHICT(NGAY);
CREATE INDEX IDX_VP_NGUON      ON VIENPHI(NGUON_THANHTOAN);
CREATE INDEX IDX_BN_BHYT       ON BENHNHAN(LOAI_BHYT);
```

### 2.4 Materialized Views cho dashboard

```sql
-- Doanh thu theo tuần
CREATE MATERIALIZED VIEW V_REVENUE_WEEKLY
REFRESH COMPLETE ON DEMAND
AS
SELECT
    TRUNC(v.NGAYTHANHTOAN, 'IW')  AS TUAN,
    k.TENKHOAPHONG,
    v.NGUON_THANHTOAN,
    SUM(v.TONGTIEN)               AS DOANHTHU,
    COUNT(DISTINCT v.MABENHNHAN)  AS SO_BENHNHAN
FROM VIENPHI v
JOIN DMKHOAPHONGBV k ON v.IDKHOAPHONG = k.IDKHOAPHONG
WHERE v.NGAYTHANHTOAN IS NOT NULL
GROUP BY TRUNC(v.NGAYTHANHTOAN, 'IW'), k.TENKHOAPHONG, v.NGUON_THANHTOAN;

-- Doanh thu theo quý
CREATE MATERIALIZED VIEW V_REVENUE_QUARTERLY
REFRESH COMPLETE ON DEMAND
AS
SELECT
    TRUNC(v.NGAYTHANHTOAN, 'Q')   AS QUY,
    k.TENKHOAPHONG,
    v.NGUON_THANHTOAN,
    SUM(v.TONGTIEN)               AS DOANHTHU,
    COUNT(DISTINCT v.MABENHNHAN)  AS SO_BENHNHAN
FROM VIENPHI v
JOIN DMKHOAPHONGBV k ON v.IDKHOAPHONG = k.IDKHOAPHONG
WHERE v.NGAYTHANHTOAN IS NOT NULL
GROUP BY TRUNC(v.NGAYTHANHTOAN, 'Q'), k.TENKHOAPHONG, v.NGUON_THANHTOAN;

-- Doanh thu theo năm
CREATE MATERIALIZED VIEW V_REVENUE_YEARLY
REFRESH COMPLETE ON DEMAND
AS
SELECT
    TRUNC(v.NGAYTHANHTOAN, 'YYYY') AS NAM,
    k.TENKHOAPHONG,
    v.NGUON_THANHTOAN,
    SUM(v.TONGTIEN)                AS DOANHTHU,
    COUNT(DISTINCT v.MABENHNHAN)   AS SO_BENHNHAN
FROM VIENPHI v
JOIN DMKHOAPHONGBV k ON v.IDKHOAPHONG = k.IDKHOAPHONG
WHERE v.NGAYTHANHTOAN IS NOT NULL
GROUP BY TRUNC(v.NGAYTHANHTOAN, 'YYYY'), k.TENKHOAPHONG, v.NGUON_THANHTOAN;

-- Refresh hàng đêm lúc 2AM (dùng DBMS_SCHEDULER)
BEGIN
  DBMS_SCHEDULER.CREATE_JOB(
    job_name   => 'REFRESH_DASHBOARD_VIEWS',
    job_type   => 'PLSQL_BLOCK',
    job_action => 'BEGIN
                     DBMS_MVIEW.REFRESH(''V_REVENUE_WEEKLY'');
                     DBMS_MVIEW.REFRESH(''V_REVENUE_QUARTERLY'');
                     DBMS_MVIEW.REFRESH(''V_REVENUE_YEARLY'');
                   END;',
    repeat_interval => 'FREQ=DAILY;BYHOUR=2;BYMINUTE=0',
    enabled    => TRUE
  );
END;
```

### 3.3 JWT implementation

```java
// JwtTokenProvider.java (tóm tắt)
@Component
public class JwtTokenProvider {

    @Value("${jwt.secret}") private String secret;
    private static final long ACCESS_EXPIRY  = 15 * 60 * 1000;       // 15 phút
    private static final long REFRESH_EXPIRY = 7 * 24 * 60 * 60 * 1000; // 7 ngày

    public String generateAccessToken(AppUser user) {
        return Jwts.builder()
            .subject(user.getUsername())
            .claim("role",     user.getRole().getRoleCode())
            .claim("khoaId",   user.getIdkhoaphong())
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + ACCESS_EXPIRY))
            .signWith(getKey())
            .compact();
    }

    public String generateRefreshToken(AppUser user) { /* ... */ }
    public Claims validateToken(String token) { /* ... */ }
    private SecretKey getKey() { return Keys.hmacShaKeyFor(secret.getBytes()); }
}
```

```java
// AuthController.java (tóm tắt)
@PostMapping("/login")
public ResponseEntity<LoginResponse> login(@RequestBody LoginRequest req, HttpServletResponse res) {
    AppUser user = authService.authenticate(req.getUsername(), req.getPassword(), req.getRole());
    String access  = jwtProvider.generateAccessToken(user);
    String refresh = jwtProvider.generateRefreshToken(user);

    // Refresh token vào HttpOnly Cookie — không lưu localStorage
    ResponseCookie cookie = ResponseCookie.from("refresh_token", refresh)
        .httpOnly(true).secure(true).path("/api/auth/refresh")
        .maxAge(Duration.ofDays(7)).sameSite("Strict").build();
    res.addHeader(HttpHeaders.SET_COOKIE, cookie.toString());

    return ResponseEntity.ok(new LoginResponse(access, user.getFullname(), user.getRole()));
}
```

### 3.4 Dashboard service với permission filter

```java
// DashboardService.java (tóm tắt)
@Service
public class DashboardService {

    @Cacheable(value = "dashboard-revenue", key = "#period + ':' + #khoaId")
    public RevenueResponse getRevenue(String period, Long khoaId, Authentication auth) {
        // Lấy role & khoaId từ JWT
        String role    = getRoleFromAuth(auth);
        Long   authKhoa = getKhoaFromAuth(auth);

        // TRUONGKHOA chỉ xem được khoa mình — không thể override bằng param
        if ("TRUONGKHOA".equals(role)) {
            khoaId = authKhoa;
        }
        // BACSI chỉ xem báo cáo cấp thấp hơn
        if ("BACSI".equals(role)) {
            throw new AccessDeniedException("Bác sĩ không có quyền xem doanh thu");
        }

        return switch (period) {
            case "week"    -> repo.getWeeklyRevenue(khoaId);
            case "quarter" -> repo.getQuarterlyRevenue(khoaId);
            default        -> repo.getYearlyRevenue(khoaId);
        };
    }
}
```

### 3.5 Cấu hình bảo mật Spring Security

```java
// SecurityConfig.java
@Configuration @EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(s -> s.sessionCreationPolicy(STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()
                .requestMatchers("/api/dashboard/**").hasAnyRole("GIAMDOC","TRUONGKHOA","BACSI")
                .requestMatchers("/api/dashboard/revenue").hasAnyRole("GIAMDOC","TRUONGKHOA")
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
            .build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12); // cost factor 12
    }
}
```

### 4.3 Axios interceptor (auto refresh token)

```typescript
// src/services/api.ts
const api = axios.create({ baseURL: '/api', withCredentials: true });

api.interceptors.request.use(config => {
  const token = useAuthStore.getState().accessToken;
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

api.interceptors.response.use(
  res => res,
  async err => {
    if (err.response?.status === 401 && !err.config._retry) {
      err.config._retry = true;
      try {
        const { data } = await axios.post('/api/auth/refresh', {}, { withCredentials: true });
        useAuthStore.getState().setAccessToken(data.accessToken);
        err.config.headers.Authorization = `Bearer ${data.accessToken}`;
        return api(err.config);
      } catch {
        useAuthStore.getState().logout();
        window.location.href = '/login';
      }
    }
    return Promise.reject(err);
  }
);
```

### 4.4 Route guard theo role

```typescript
// src/components/layout/ProtectedRoute.tsx
const ROLE_ROUTES: Record<string, string[]> = {
  GIAMDOC:    ['/dashboard', '/analytics', '/finance', '/pharmacy', '/reports', '/layout'],
  TRUONGKHOA: ['/dashboard', '/reports', '/layout'],
  BACSI:      ['/reports'],
};

export function ProtectedRoute({ children, path }: Props) {
  const { user, role } = useAuthStore();
  if (!user) return <Navigate to="/login" />;
  if (!ROLE_ROUTES[role]?.includes(path)) return <Navigate to="/unauthorized" />;
  return <>{children}</>;
}
```

## 5. Security

### 5.1 Các lớp bảo vệ

```
Request → HTTPS/TLS
       → CORS whitelist (chỉ domain frontend)
       → Rate Limit (10 req/s per IP trên /auth/login)
       → JWT Signature validate
       → Role check (@PreAuthorize)
       → Department filter (trong service layer)
       → Audit log ghi lại
```

### 5.2 Chính sách mật khẩu

- BCrypt với cost factor 12
- Tối thiểu 8 ký tự, có chữ hoa + số + ký tự đặc biệt
- Bắt buộc đổi mật khẩu lần đăng nhập đầu tiên
- Khoá tài khoản sau 5 lần sai liên tiếp (mở khoá sau 30 phút hoặc do admin)
- Refresh token rotate mỗi lần dùng (phát hiện token theft)

### 5.3 Các header bảo mật (Spring Boot)

```java
http.headers(h -> h
    .contentSecurityPolicy(csp -> csp.policyDirectives(
        "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'"))
    .frameOptions(f -> f.deny())
    .xssProtection(x -> x.headerValue(XXssProtectionHeaderWriter.HeaderValue.ENABLED_MODE_BLOCK))
    .httpStrictTransportSecurity(hsts -> hsts
        .includeSubDomains(true).maxAgeInSeconds(31536000))
);
```

## 6. Performance

### 6.1 Redis cache strategy

```
Key pattern:     dashboard-revenue:{period}:{khoaId}
TTL:             5 phút (300 giây)
Invalidate khi:  Admin trigger manual refresh hoặc sau daily job

Key pattern:     dashboard-kpi:{period}:{khoaId}
TTL:             5 phút

Không cache:     Dữ liệu bệnh nhân cụ thể (PII sensitive)
                 Layout prefs (per-user, đọc nhanh từ DB)
```

### 6.2 Oracle indexes

```sql
-- Đã có trong schema:
IDX_VIENPHI_TIME    ON VIENPHI(NGAYTHANHTOAN)
IDX_VIENPHI_KHOA    ON VIENPHI(IDKHOAPHONG)
IDX_CT_VP           ON VIENPHICT(IDVIENPHI)
IDX_CT_DV           ON VIENPHICT(IDGIAVIENPHI)
IDX_CT_THUOC        ON VIENPHICTTHUOC(IDVIENPHI)

-- Cần thêm:
IDX_CT_KHOA         ON VIENPHICT(IDKHOAPHONG)
IDX_CT_NGAY         ON VIENPHICT(NGAY)
IDX_VP_NGUON        ON VIENPHI(NGUON_THANHTOAN)
IDX_COMPOSITE_1     ON VIENPHI(IDKHOAPHONG, NGAYTHANHTOAN)  -- compound
```

### 6.3 Frontend performance

- **Code splitting**: mỗi page lazy load riêng với `React.lazy()`
- **React Query**: cache API response ở client 5 phút, stale-while-revalidate
- **Chart.js**: destroy và re-init chart khi đổi loại — tránh memory leak
- **Debounce**: filter khoa dropdown debounce 300ms trước khi gọi API
- **Vite build**: tree-shaking, chunk splitting, gzip compression

      SELECT 'REVENUE',        'READ'      FROM DUAL UNION ALL
